# Modding Resources

## API Reference: dump.cs.txt

The sole API reference is `dump.cs.txt` at the repo root (gitignored). Generated by Il2CppDumper from the game's compiled assemblies. ~917K lines, 34 MB, containing 6,335 class definitions.

### How to Use
- Search with grep for type or method names: `grep -n "class MyType" dump.cs.txt`
- Definitions include fields, properties, methods, and inheritance but NOT method bodies
- Organized by Image (assembly DLL) then by Namespace

### Key Types for Modding

**UI Framework:**
- `ElementObjectManager` — Core UI element system. Use `serializedElements[]` array (not `GetLabels()` which fails in Il2Cpp interop).
- `BaseMenuViewController` — Base screen controller. Access UI via `m_View` property.
- `SelectionButton` / `SelectionItem` — Interactive button classes. `isSelected` for focus state.
- `ViewControllerManager` — Manages VC navigation stack. `GetFocusViewController()` for current screen.
- `DialogViewControllerManager` — Singleton extending ViewControllerManager for dialogs.
- `ActionSheetViewController` — Multi-choice dialog. Uses `InfinityScrollView` for lists.

**Duel:**
- `DuelClient` — Main duel controller
- `DuelLP` — Life point management with `ChangeLP()` method
- `CardRoot` — Card instance with `isFace`, `team`, `cardLocator.pos`

## Il2Cpp Interop Notes

- Cast game objects: `.Cast<T>()` / `.TryCast<T>()` from Il2CppInterop.Runtime
- Register custom types: `ClassInjector.RegisterTypeInIl2Cpp<T>()`
- Harmony: Only Prefix/Postfix patches work (no Transpiler)
- Il2Cpp collections don't always interop cleanly (e.g., `SortedDictionary.KeyCollection` from `GetLabels()`)
